## Phase 2: Bot API 与策略服务

### 目标

- 起一个独立 `bot-api`
- 提供 bot 配置、心跳、决策、日志接口
- 提供策略模板和赚钱项开关
- 提供 bot 授权、套餐与 entitlement 校验

### 建议接口

- `GET /api/bots/:wallet`

- `POST /api/bots/:wallet/heartbeat`
- `POST /api/bots/:wallet/decision`
- `POST /api/bots/:wallet/telemetry`
- `GET /api/strategies`
- `GET /api/entitlements/:wallet`
- `POST /api/billing/webhook`
- `GET /api/plans`

### 当前仓库映射

- `apps/aiBot`
  - 当前可继续承接最小 Bot API / BFF / 管理壳
- `apps/aiBrainService`
  - 当前可继续承接 Strategy / Planner / Replay / Risk

## Phase 3: Token Task Bot MVP

### 目标

- 打通第一条赚钱闭环

### 动作链

1. 拉奖池状态
2. 计算收益分
3. 自动接任务
4. 自动推进任务
5. 自动领奖
6. 上报收益

### 验收标准

- 能连续稳定跑
- 能正确计算单次收益
- 能在奖励不足时停止

## Phase 4: Farming / Harvest Bot

### 目标

- 打通第二条赚钱闭环

### 动作链

1. 扫描成熟点
2. 自动收获
3. 检查库存
4. 自动补循环
5. 无资源则切换策略

### 验收标准

- 客户端执行链稳定
- 行为不干扰玩家正常操作
- 收益有统计

## Phase 5: Equipment Recovery Bot

### 目标

- 打通第三条赚钱闭环

### 动作链

1. 资产价值估算
2. 白名单/黑名单过滤
3. 执行回收
4. 领取额外奖励
5. 记录实际收益

### 验收标准

- 不误回收高价值物品
- 可回放每次回收决策
- 链上失败可解释

## Phase 6: AI Advisor 与回测系统

### 目标

- 用历史数据回放策略
- 引入 AI 决策解释
- 支持参数自动调优

### 依赖数据

- `service-xlsx-tool-v2` 的静态表
- Unity 侧采集回传的任务与行为日志
- `theweb3-v2` 的链上池子与 rollup 结果
- `contracts-monad-v2` 的规则约束

## Phase 7: 风控与灰度

### 目标

- 可配置风险策略
- entitlement 过期后的降级与宽限控制
- 异常高成本或不可逆动作的额外校验
- 连续异常失败时的降级 / 切策略 / 暂停机制
- 人工接管入口与必要让权
- 默认放开低风险常规收益动作

## 推荐 MVP 路径

按权威文档，最快落地顺序是：

1. `Unity AIBot 面板 + BotRuntime 空壳`
2. `Bot API`
3. `Token Task Bot`
4. `Harvest Bot`
5. `Equipment Recovery Bot`
6. `AI 决策增强`
