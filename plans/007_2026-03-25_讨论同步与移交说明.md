### 3. 当前正式主线

- 产品定位：

- Unity 执行术语：
  - 统一使用 `操作原子(Atoms)`，不再使用 `Tool Catalog / *Tools`
- 数据边界：
  - `client-observable-first`
- 收费方向：
  - `按 Bot 收费`
- 介入原则：
  - 玩家默认只做必要介入，不做全程审批式控制

### 4. 当前版本操作原子边界

- 当前版本讨论的 `操作原子(Atoms)`，只锁定在：
  - `角色已经登录`
  - `角色已经进入游戏场景`
  - `Bot 在游戏内执行游玩行为`
- 当前版本 **不纳入** 操作原子范围的内容：
  - `登录`
  - `创角`
  - `切服`
  - `钱包连接 / 钱包签名`
  - `链上资产搬运`
  - `队伍管理 / 社交管理 / 聊天`
  - `纯配置页 / 设置页 / 非游玩态管理动作`
- 因此，当前版本的操作原子不是“全系统所有按钮”，而是“角色进入游戏后，为完成玩法循环所需的最小执行原子”。

### 5. 当前版本操作原子分类原则

- `操作原子` 的定义：
  - 指 Unity Client 在游戏内可主动发起、可组合、可观察结果、可用于 AI 决策执行的最小动作单元。
- `操作原子` 不等于：
  - 页面按钮
  - 管理后台接口
  - 大而全的玩法脚本
  - 登录/创角/钱包/社交这类非当前版本执行范围的动作
- 原子拆分原则：
  - 一个原子只做一个明确动作
  - 一个原子只负责一个主要成功/失败结果
  - 能复用现有 Unity Client 真实入口时，优先对齐真实入口，不另造抽象名词
  - 对于家园、种植、农牧这类玩法，允许多个原子共同组成完整循环，不把一整条循环包装成单一黑盒原子
  - 如果 Unity 代码里已经存在明确分叉，就不能继续用一个宽泛原子混写
- 当前版本特别说明：
  - `随机移动` 不是独立底层网络包，而是 `随机可达点选择策略 + 按坐标移动` 的组合
  - `按目标移动` 不能只写成一个泛化动作，至少要区分：
    - `按场景实体目标接近`
    - `按种植土地目标接近`
    - 其中 `NPC / 怪物 / 采集物 / 畜牧动物 / 普通可交互实体` 可统一视为 `场景实体`
  - `复活` 不能写成单一原子，当前 Unity 侧至少已经明确分成：
    - `原地复活`
    - `回城复活`
    - `放弃复活`
  - `场景种植/畜牧/采集目标选择` 不能只写成“选择可交互物”，当前目标类型至少区分：
    - `Soil`
    - `Resource`
    - `Animal`
  - `AnimalBowl` 先不纳入当前版本原子范围，它属于旧 `home` 逻辑遗留，不作为这一版正式原子
  - 当前 Unity 代码仍保留 `HomeSoil / HomeAnimal / HomeResources` 等历史类名，但在产品语义上应理解为：
    - `场景中的种植土地`
    - `场景中的畜牧动物`
    - `场景中的采集物`

### 6. 当前版本操作原子正式分类

- `MoveAtoms`：移动与寻路原子。
  - `move_to_position`: 按明确坐标点移动，用于寻路到指定位置。
  - `move_to_scene_entity_target`: 朝场景实体目标接近移动，目标可为 NPC、怪物、采集物、畜牧动物、普通可交互实体等。
  - `move_to_soil_target`: 朝场景中的种植土地接近移动。
  - `stop_move_and_sync`: 停止主动移动并向服务端同步当前位置状态。
  - `random_roam_move`: 先生成随机可达点，再执行按坐标移动的游荡行为。
  - 其中 `random_roam_move` 属于 Bot 选点策略原子，最终仍落到 `move_to_position`
- `TravelAtoms`：场景切换原子。
  - `change_scene_to_world`: 切换回大世界场景。
  - `change_scene_to_dungeon`: 切换进入目标副本场景。
- `RespawnAtoms`：复活原子。
  - `respawn_local`: 倒计时结束后在本地原地复活。
  - `respawn_city`: 死亡后选择回主城区复活。
  - `respawn_give_up`: 死亡后选择放弃当前局面并执行系统定义的退出复活。
- `TargetAcquireAtoms`：目标获取原子。
  - `acquire_combat_target`: 搜索并锁定战斗目标。
  - `acquire_interact_target`: 搜索并锁定通用交互目标。
  - `acquire_soil_target`: 搜索并锁定场景中的种植土地目标。
  - `acquire_resource_target`: 搜索并锁定场景中的采集物目标。
  - `acquire_animal_target`: 搜索并锁定畜牧动物目标。
- `CombatAtoms`：战斗执行原子。
  - `approach_attack_range`: 向目标移动到可攻击距离。
  - `cast_skill_on_target`: 对指定目标释放技能。
  - `cast_skill_on_position`: 对指定地面位置释放技能。
  - `continue_combat`: 在当前战斗上下文中持续执行攻击循环。
- `ResourceGatherAtoms`：资源采集原子。
  - `interact_world_resource`: 对场景中的采集物资源点发起交互。
  - `gather_target`: 对当前采集目标执行采集动作。
  - `pickup_drop`: 拾取地面掉落物。
  - `collect_ground_result`: 收集采集或战斗后落地的结果物。
  - `general_entity_interact`: 对普通实体执行非战斗型交互。
- `EnergyAtoms`：精力相关原子。
  - `buy_energy_supply`: 在游戏内购买精力补给或精力药水。
- `ConsumableAtoms`：消耗品使用原子。
  - `use_consumable_item`: 对目标实体使用可消耗道具的统一入口。
  - `use_energy_potion`: 使用恢复精力类道具。
  - `use_hp_potion`: 使用恢复生命类道具。
  - `use_buff_potion`: 使用增益类药剂或 Buff 道具。
- `EquipmentAtoms`：装备穿脱原子。
  - `equip_avatar_item`: 给玩家或宠物穿戴装备。
  - `unequip_avatar_item`: 脱下当前已穿戴功能性装备。
- `DialogueAtoms`：对话交互原子。
  - `entity_dialogue`: 与场景实体打开通用对话，不限于任务 NPC。
- `talentAtoms`：天赋成长原子。
  - `upgrade_talent_node`: 升级天赋树节点。
  - `reset_talent_layer`: 重置指定天赋系。
  - `update_user_skill_group`: 更新玩家技能组配置。
- `PlantingAtoms`：种植原子。
  - `hoe_soil`: 对土地执行锄地整理。
  - `water_soil`: 对作物土地执行浇水。
  - `harvest_crop`: 对成熟作物执行收获。
- `HusbandryAtoms`：畜牧原子。
  - `appease_animal`: 对动物执行安抚，提高可操作状态。
  - `harvest_animal_product`: 收取动物产出的资源。
  - `touch_dead_animal`: 对死亡动物执行触摸处理。
  - `free_animal`: 对动物执行放生。
  - `make_pet_egg`: 触发宠物产蛋流程。
- `TaskAtoms`：普通任务原子。
  - `npc_task_dialogue`: 与 NPC 打开任务对话。
  - `accept_normal_task`: 接取普通任务。
  - `abandon_normal_task`: 放弃普通任务。
  - `accept_task_chain`: 接取任务链。
  - `abandon_task_chain`: 放弃任务链。
  - `submit_task_progress`: 提交任务进度或任务物品。
  - `receive_task_reward`: 领取任务奖励。
- `TokenTaskAtoms`：Token 任务原子。
  - `refresh_token_task_pool`: 刷新 Token Task Pool 候选任务。
  - `accept_token_task`: 接取 Token 任务。
  - `submit_token_task_progress`: 提交 Token 任务进度。
  - `receive_token_task_reward`: 领取 Token 任务奖励。
  - `abandon_token_task`: 放弃当前 Token 任务。
- `EquipmentStakeAtoms`：装备质押原子。
  - `query_stake_nft_list`: 查询当前已质押装备列表。
  - `stake_equipment_nfts`: 将背包中的装备执行质押 / 绑定。
  - `query_stake_nft_record`: 查询装备质押或熔炼记录。
  - `smelt_staked_nfts`: 对已质押装备执行批量熔炼。
- `FishingAtoms`：钓鱼原子。
  - `pull_rod`: 执行拉杆操作。
  - `fishing_battle_qte`: 执行钓鱼过程中的 QTE 对抗。
  - `exit_fishing`: 主动退出钓鱼状态。
  - `throw_rock`: 在钓鱼过程中执行丢石头动作。
- `DungeonAtoms`：副本原子。
  - `enter_world_dungeon_portal`: 从大世界入口进入副本。
  - `complete_instancing_level`: 完成当前副本关卡并结算。
  - `exit_instancing_level`: 主动退出当前副本关卡。

### 7. 当前版本明确排除的原子分类

- `LoginAtoms`：登录原子。
  - 例如 `login_game`、`create_player`
- `WalletAtoms`：钱包与链上原子。
  - 例如 `move_nft_to_onchain`、`move_onchain_nft_to_game`
- `DeprecatedHomeAtoms`：沿旧 `home` 播种链路的遗留原子。
  - `sow_seed` 当前版本不实现；场景种植链路只保留 `hoe_soil / water_soil / harvest_crop`
- `TeamAtoms`：组队原子。
  - 例如建队、邀人、踢人、转队长、退队
- `SocialAtoms`：社交原子。
  - 例如聊天、招募广播、纯社交交互
- `SettingAtoms`：设置原子。
  - 例如设置保存、UI 偏好、非玩法态配置动作
- `HandItemAtoms`：手持槽原子。
  - 当前版本不实现手持槽装卸链路
- `InventoryAtoms`：库存管理原子。
  - 当前版本不实现库存转移、主动丢弃与背包快照刷新链路
- `EquipmentRefineAtoms`：装备强化原子。
  - 当前版本不实现装备强化、继承、拆解与结果提取链路
- `EscortAtoms`：押镖原子。
  - 当前版本不实现押镖链路
- `RewardAtoms`：奖励领取原子。
  - 当前版本不实现爬塔抽奖、券类抽奖与积分排行领奖链路
- `StorageAtoms`：仓储原子。
  - 当前版本不实现仓库租用与库存扩展链路
- `CampAtoms`：阵营原子。
  - 当前版本不实现阵营切换链路
- `BuildAtoms`：建筑原子。
  - 当前版本不实现建筑放置、充能、收取、回收与电池生成链路
- `TotemAtoms`：图腾原子。
  - 当前版本不实现图腾放置、激活、补能、产出收取、回收、报价与传送链路

### 8. 当前版本原子落地规则

这一节只回答 4 个问题：当前版本的原子边界是什么、应该怎么拆、哪些保留、哪些排除。

边界：

- Brain 输出给 Unity Runtime 的执行指令，只能引用当前版本允许的原子分类。
- 只有“角色进入游戏后，在场景内游玩所需的执行动作”才属于当前版本原子。
- 如果某动作属于登录、钱包、队伍、社交、设置，或当前版本明确不做的玩法域，就不进入当前版本原子目录。

拆分原则：

- 一个玩法循环如果需要多个步骤完成，必须拆成多个原子，不能写成单一“大原子”。
- 场景种植、场景畜牧、资源采集、打怪，优先按 Unity Client 已存在的真实动作入口做原子映射。
- `TargetAcquireAtoms` 属于执行前置原子，不能省略成隐式黑盒。

当前版本保留的抽象方式：

- 移动原子不再允许统一写成 `NavigationAtoms` 单桶，至少拆成 `MoveAtoms`、`TravelAtoms`、`RespawnAtoms`。
- 当前版本的核心移动原子收敛为 `move_to_scene_entity_target` 与 `move_to_soil_target`。
- `NPC / 怪物 / 采集物 / 畜牧动物 / 普通交互实体` 的接近移动，统一归入 `move_to_scene_entity_target`。
- `respawn_local / respawn_city / respawn_give_up` 必须分开，因为 Unity 已存在不同入口与不同结果。
- `random_roam_move` 只作为 Bot 策略层原子存在，执行时必须最终降解为 `move_to_position`。

当前版本明确排除：

- `AnimalBowl` 与 `move_to_animal_bowl_target / acquire_animal_bowl_target / fill_animal_bowl` 不纳入当前版本正式原子，因为它属于旧 `home` 逻辑遗留。
- `teleport_to_position` 不纳入当前版本正式原子，因为它属于测试接口，不作为正式游玩执行入口。
- `loot_after_combat` 不纳入当前版本正式原子，因为当前实体掉落战利品直接进入背包，不需要单独拾取原子。
- `borrow_energy / repay_energy_debt` 不纳入当前版本正式原子，因为当前版本不实现精力借贷与还债链路。
- `use_pet_feed_item` 不纳入当前版本正式原子，因为当前版本不实现宠物喂食道具链路。
- `equip_appearance_item` 不纳入当前版本正式原子，因为第一版本不考虑时装 / 外观装备链路。
- `HandItemAtoms / InventoryAtoms / EquipmentRefineAtoms / EscortAtoms / RewardAtoms / StorageAtoms / CampAtoms` 不纳入当前版本正式原子，因为这些执行域当前版本不实现。
- `BuildAtoms / TotemAtoms / world_totem_teleport` 不纳入当前版本正式原子，因为这些执行域当前版本不实现。

文档归属：

- 当前版本的详细原子清单、实现形态、优先级与执行规格，统一下沉到 [plans/008_游戏内操作原子详细清单.md](/Users/44alex/work/meland/odyssey-v2/lumiGameBot/plans/008_游戏内操作原子详细清单.md)。
- 后续如果要扩大到登录、钱包、队伍、社交等域，应先补版本边界说明，再进入新的原子清单，而不是直接混入当前目录。
